# Idle RPG 프로젝트 청사진

이 문서는 Idle RPG 프로젝트의 개발 계획, 아키텍처, 디자인 원칙 및 구현 단계를 기록하는 중앙 허브 역할을 합니다.

## 1. 애플리케이션 개요

**Idle RPG**는 사용자가 최소한의 조작으로 캐릭터를 성장시키는 방치형 웹 게임입니다. 사용자는 Google 계정으로 로그인하고, 캐릭터는 자동으로 몬스터와 싸우며 성장합니다. 이 프로젝트는 Cloudflare Pages에 배포되며, 정적 프론트엔드와 서버리스 Functions API를 결합한 현대적인 웹 아키텍처를 사용합니다.

## 2. 핵심 기능

- **Google 계정 기반 인증:** Cloudflare Pages Functions를 이용한 안전한 OAuth 2.0 로그인.
- **자동 전투 시스템:** 사용자가 보고 있지 않아도 캐릭터는 계속해서 전투하고 재화를 획득합니다.
- **캐릭터 성장:** 전투를 통해 경험치와 골드를 획득하여 레벨업하고 능력치를 강화합니다.
- **실시간 데이터 동기화:** Cloudflare D1 데이터베이스를 사용하여 모든 사용자 데이터를 저장하고 동기화합니다.

## 3. 기술 스택

- **프론트엔드:** 순수 HTML, CSS, JavaScript (프레임워크 없음)
- **백엔드 (API):** Cloudflare Pages Functions (JavaScript)
- **데이터베이스:** Cloudflare D1 (SQLite 기반 서버리스 데이터베이스)
- **인증:** Google OAuth 2.0
- **배포:** Cloudflare Pages
- **개발 환경:** Firebase Studio IDE

## 4. 구현 및 디버깅 기록

이 섹션은 프로젝트 개발 중 발생했던 주요 문제들과 해결 과정을 시간 순서대로 기록합니다.

### 4.1. 기능 구현: Google OAuth 로그인 (Cloudflare 표준 방식)
- **목표:** 사용자가 Google 계정으로 안전하게 로그인할 수 있는 기능을 구현합니다.
- **초기 문제:** 복잡한 `_middleware.js` 파일이 Cloudflare의 기본 라우팅 시스템과 충돌하여 API 요청 시 404 오류가 발생했습니다.
- **해결:**
  - `_middleware.js`를 제거하고, 각 API 엔드포인트가 독립적으로 작동하도록 코드를 재구성했습니다.
  - `functions/api/login.js`: 사용자를 Google 로그인 페이지로 리디렉션하는 단순 함수.
  - `functions/api/callback.js`: Google로부터 인증 코드를 받아 사용자 정보를 처리하는 단순 함수.
  - 이 변경을 통해 Cloudflare Pages의 파일 기반 라우팅을 정상적으로 사용하게 되었습니다.

### 4.2. 배포 플랫폼 문제 디버깅 (Worker vs. Pages 프로젝트 타입)
- **증상:** 배포 후 `*.workers.dev` 주소에서 계속해서 `404 Not Found` 오류 발생. `Settings` 메뉴에 `Functions` 하위 메뉴와 `Environment variables` 추가 기능이 보이지 않음.
- **근본 원인:** Cloudflare에서 프로젝트가 **"Functions를 포함한 풀스택 사이트"**가 아닌, API 기능이 없는 **"단순 정적 사이트(Static Site)"**로 잘못 분류되었음. 이로 인해 `functions` 폴더의 모든 코드가 배포에서 제외됨.
- **최종 해결 조치:**
  1. 기존의 잘못된 설정을 가진 프로젝트를 Cloudflare 대시보드에서 완전히 **삭제**했습니다.
  2. GitHub 저장소를 연결하여 **Pages** 프로젝트를 **재생성**했습니다.
  3. 프로젝트 생성 시, **`Build output directory`를 `public`으로** 명시적으로 지정하고, **`Framework preset`을 `None`**으로 설정하여 Cloudflare가 `functions` 폴더를 올바르게 인식하도록 유도했습니다.
  4. 환경 변수(`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`)를 Cloudflare Pages 설정에 안전하게 등록했습니다.
  5. 배포 후, `idlerpg.pages.dev` 라는 새로운 주소를 부여받아 모든 기능이 정상 작동하는 것을 확인했습니다.

### 4.3. 기능 구현: Cloudflare D1 데이터베이스 연동
- **목표:** 사용자의 계정 정보, 캐릭터 데이터, 세션을 저장하고 관리합니다.
- **구현:**
  1. Cloudflare 대시보드에서 `idlerpg_db` D1 데이터베이스를 생성했습니다.
  2. `users` 테이블 (캐릭터 정보 저장)과 `sessions` 테이블 (로그인 상태 유지) 스키마를 정의하고 생성했습니다.
  3. `idlerpg` Pages 프로젝트 설정에서 `DB`라는 이름으로 D1 데이터베이스 바인딩을 추가했습니다.
  4. `callback.js`를 수정하여 로그인 시 `users` 테이블에서 사용자를 조회하거나 새로 생성하고, `sessions` 테이블에 세션 정보를 기록하도록 구현했습니다.
  5. 브라우저에 `session_id`를 쿠키로 설정하여 로그인 상태를 유지하도록 했습니다.
  6. `/api/me` 엔드포인트를 추가하여, 쿠키를 검증하고 로그인된 사용자의 모든 데이터를 DB에서 불러와 프론트엔드로 전달하는 기능을 구현했습니다.
  7. 프론트엔드 `main.js`에서 페이지 로드 시 `/api/me`를 호출하여 로그인 상태에 따라 UI를 동적으로 변경하도록 수정했습니다.

## 5. 향후 개발 계획: 자동 전투 시스템

이 섹션은 앞으로 구현할 자동 전투 시스템의 세부 요구사항을 정의합니다.

- **17. 스테이지 개념 추가:** 게임의 진행 단계를 스테이지로 구분합니다.
- **18. 몬스터 스탯 동적 적용:** 스테이지 레벨에 따라 몬스터의 체력, 공격력, 방어력 등이 동적으로 결정됩니다.
- **19. DPS 기반 전투 로직:** 캐릭터의 초당 피해량(DPS) 공식에 따라 몬스터와의 전투를 시뮬레이션합니다.
- **20. 스테이지 클리어 및 진행:** 한 스테이지의 모든 몬스터(예: 10마리)를 처치하면 클리어 보상(골드)을 받고, 자동으로 다음 스테이지로 넘어갑니다.
- **21. 캐릭터 사망 처리:** 캐릭터의 체력이 0이 되면 사망 처리되며, 잠시 후 부활하여 이전 스테이지에서 다시 시작합니다. (사망 페널티)
- **22. 오프라인 보상:** 사용자가 접속하지 않은 시간과 마지막으로 클리어한 스테이지를 기준으로 계산된 골드를 보상으로 지급합니다.
- **23. 스테이지 진행률 UI:** 현재 스테이지의 진행 상황을 보여주는 ProgressBar를 UI에 표시합니다.
- **24. 몬스터 HP UI:** 현재 전투 중인 몬스터의 남은 체력을 보여주는 HP Bar를 UI에 표시합니다.
- **25. 현재 DPS 표시:** 캐릭터의 현재 DPS 수치를 화면에 표시합니다.
- **26. 숫자 단위 축약:** 골드, 데미지 등의 큰 숫자를 K(천), M(백만), B(십억), T(조) 등으로 축약하여 표시합니다.
- **27. 치트 방지:** 클라이언트에서 조작된 전투 결과를 서버에서 검증하는 로직을 구현합니다.
- **28. 밸런싱 자동화:** 게임 밸런스(캐릭터 성장 곡선, 몬스터 스탯 등) 조정을 위한 도구나 시스템을 고려합니다.
- **29. 주기적 서버 동기화:** 약 30초마다 클라이언트의 게임 상태(현재 스테이지, 골드 등)를 서버와 자동으로 동기화하여 데이터 정합성을 유지합니다.
- **30. 오프라인 보상 팝업:** 로그인 시 오프라인 보상이 있을 경우, 팝업 창을 통해 보상 내역을 보여주고 수령할 수 있도록 합니다.
- **31. 캐릭터 사망 연출:** 캐릭터 사망 시 화면에 시각적인 효과를 추가하여 사용자에게 알립니다.

### 5.1. 1단계 구현 계획: 핵심 전투 루프 및 UI
- **목표:** 클라이언트 사이드에서 전투가 시뮬레이션되고, 그 과정이 사용자에게 시각적으로 보이도록 하는 기본 틀을 구축합니다.
- **주요 작업:**
  - `index.html`: 전투 관련 UI 요소(스테이지 진행바, 몬스터 HP바 등) 추가.
  - `main.js`: 핵심 게임 루프를 생성하고, DPS 계산 및 UI 업데이트 로직 구현.
  - 서버 API는 다음 단계에서 구현하며, 우선 클라이언트에서 모든 로직을 시뮬레이션합니다.
