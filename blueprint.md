# Idle RPG 프로젝트 청사진

이 문서는 Idle RPG 프로젝트의 개발 계획, 아키텍처, 디자인 원칙 및 구현 단계를 기록하는 중앙 허브 역할을 합니다.

## 1. 애플리케이션 개요

**Idle RPG**는 사용자가 최소한의 조작으로 캐릭터를 성장시키는 방치형 웹 게임입니다. 사용자는 Google 계정으로 로그인하고, 캐릭터는 자동으로 몬스터와 싸우며 성장합니다. 이 프로젝트는 Cloudflare Pages에 배포되며, 정적 프론트엔드와 서버리스 Functions API를 결합한 현대적인 웹 아키텍처를 사용합니다.

## 2. 핵심 기능

- **Google 계정 기반 인증:** Cloudflare Pages Functions를 이용한 안전한 OAuth 2.0 로그인.
- **자동 전투 시스템:** 사용자가 보고 있지 않아도 캐릭터는 계속해서 전투하고 재화를 획득합니다.
- **캐릭터 성장:** 전투를 통해 경험치와 골드를 획득하여 레벨업하고 능력치를 강화합니다.
- **실시간 데이터 동기화:** Cloudflare D1 데이터베이스를 사용하여 모든 사용자 데이터를 저장하고 동기화합니다.

## 3. 기술 스택

- **프론트엔드:** 순수 HTML, CSS, JavaScript (프레임워크 없음)
- **백엔드 (API):** Cloudflare Pages Functions (JavaScript)
- **데이터베이스:** Cloudflare D1 (SQLite 기반 서버리스 데이터베이스)
- **인증:** Google OAuth 2.0
- **배포:** Cloudflare Pages
- **개발 환경:** Firebase Studio IDE

### 4.6. UI/UX 개편: 확장성을 위한 탭 기반 레이아웃
- **목표:** 향후 추가될 장비, 레이드, 상점 등 다양한 콘텐츠를 담을 수 있는 확장 가능한 UI 구조를 만듭니다.
- **구현 계획:**
    1. **HTML 구조 변경 (`index.html`):**
        - 게임 뷰 내에 상단 네비게이션 메뉴 영역(`<nav>`)을 생성합니다.
        - "상태", "장비", "레이드" 버튼을 메뉴에 추가합니다.
        - 각 메뉴에 해당하는 콘텐츠 패널(`status-view`, `equipment-view`, `raid-view`)을 생성합니다. 기존의 캐릭터 정보 표시는 `status-view`로 이전됩니다.
    2. **CSS 스타일링 (`style.css`):**
        - 상단 메뉴 바에 대한 스타일을 추가하여 다른 콘텐츠와 시각적으로 구분합니다.
        - 탭 버튼 스타일을 디자인하고, 현재 활성화된 탭을 명확하게 표시하는 `:active` 또는 클래스 기반 스타일을 추가합니다.
        - 전체적인 레이아웃을 다듬어 일관성 있고 깔끔한 디자인을 유지합니다.
    3. **JavaScript 기능 구현 (`main.js`):**
        - 메뉴 버튼에 클릭 이벤트 리스너를 추가합니다.
        - 버튼 클릭 시, 해당 콘텐츠 패널만 표시하고 나머지 패널은 숨기는 로직을 구현합니다. (예: `hidden` 클래스 토글)
        - 페이지 로드 시 기본으로 '상태' 탭이 선택되도록 설정합니다.
