# Idle RPG 프로젝트 청사진

## 1. 프로젝트 개요

**목표:** 웹 기반 방치형 RPG 게임 개발
**핵심 기능:**
- Google 계정 기반 인증
- 자동 전투 및 재화 획득
- 캐릭터 스탯 성장 시스템
- 스테이지 진행 및 몬스터 조우
- 미려하고 직관적인 UI/UX

- **프론트엔드:** HTML, CSS, JavaScript (프레임워크 없음)
- **백엔드:** Cloudflare Workers (Functions)
- **데이터베이스:** Cloudflare D1 (SQLite 기반)
- **인증:** Google OAuth 2.0

### 5.4. 게임 화면 UI/UX 전면 개편

**목표:** 사용자가 게임의 핵심 정보(전투, 성장)를 한눈에 파악하고, 미적으로 만족스러운 경험을 할 수 있도록 메인 게임 화면의 UI/UX를 현대적인 디자인으로 전면 개편합니다.

**세부 개발 계획:**

1.  **HTML 구조 변경 (`index.html`):**
    *   기존의 탭 기반 뷰 전환 구조(`game-menu`, `.view`)를 제거합니다.
    *   `game-view` 내부에 `main-content` 영역을 생성하고, 이를 2단 그리드 레이아웃으로 설정합니다.
    *   왼쪽 열에는 전투 관련 UI(`battle-screen`)를 배치합니다.
    *   오른쪽 열에는 캐릭터 정보 및 성장 UI(`character-panel`)를 배치합니다.
    *   사용자 정보(`user-info`)와 로그아웃 버튼 등을 포함하는 헤더(`game-header`)를 간소화하여 상단에 배치합니다.

2.  **CSS 스타일 재정의 (`style.css`):**
    *   **전체 레이아웃:** `body`의 스타일을 조정하여 게임 컨테이너가 화면 중앙에 위치하도록 합니다.
    *   **2단 그리드:** `main-content`에 `display: grid`와 `grid-template-columns`를 적용하여 좌우 2단 레이아웃을 구현합니다.
    *   **카드 디자인:** `battle-screen`과 `character-panel`을 시각적인 '카드'로 디자인합니다. `background-color`, `border-radius`, `box-shadow`를 적용하여 입체감을 부여합니다.
    *   **컴포넌트 스타일 개선:** 몬스터 정보, 진행도 바, 스탯 항목, 버튼 등 모든 UI 컴포넌트의 디자인(색상, 폰트, 여백, 정렬)을 개선하여 시각적 통일성과 완성도를 높입니다.
    *   **반응형 디자인:** 화면 폭이 좁아지는 모바일 환경에서는 2단 레이아웃이 1단으로 자연스럽게 변경되도록 미디어 쿼리를 적용합니다.

3.  **JavaScript 로직 수정 (`main.js`):**
    *   기존의 탭 메뉴 클릭 이벤트 리스너와 뷰 전환 로직(`showGameView`, `showLoginView` 일부)을 제거합니다.
    *   UI 요소에 대한 참조 변수(DOM Selector)를 새로운 HTML 구조에 맞게 업데이트합니다.
    *   UI 업데이트 함수(`updateAllUI`, `updateCharacterUI` 등)는 그대로 유지하되, 변경된 요소에 정확히 바인딩되도록 합니다.
